# Mankaew — อ่านก่อนเริ่ม (สำหรับเพื่อนร่วมทีม) 👋

อยากให้เพื่อนเข้าใจโปรเจกต์นี้ใน 5 นาที — อ่านไฟล์เดียวจบ

---

## โปรเจกต์นี้คืออะไร

**Mankaew** = เว็บกดบัตรคอนเสิร์ต/เทศกาล (คล้าย Eventbrite / TicketMelon)
มีผู้ใช้ 3 แบบ:

| ใคร | ทำอะไร |
|---|---|
| 🎟️ **คนซื้อบัตร** (Buyer) | หาคอนเสิร์ต → เข้าคิว → เลือกที่นั่ง → จ่ายเงิน → ได้ตั๋ว QR |
| 🎤 **คนจัดงาน** (Organizer) | สร้างอีเวนต์, ดูยอดขาย, สแกนตั๋วหน้างาน |
| 🛡️ **แอดมิน** (Admin) | ดูภาพรวมทั้งระบบ, อนุมัติอีเวนต์, จัดการเงิน/คืนเงิน |

---

## ตอนนี้ทำอะไรไปแล้ว

**หน้าจอ (UI) เสร็จครบทั้ง 3 ฝั่งแล้ว** — รวม 21 หน้า

- ฝั่งคนซื้อ: 10 หน้า (หน้าแรก, รายการอีเวนต์, รายละเอียด, ห้องรอคิว, เลือกที่นั่ง, ตะกร้า, จ่ายเงิน, ตั๋ว, บัญชี, ล็อกอิน)
- ฝั่งคนจัดงาน: 6 หน้า (แดชบอร์ด, อีเวนต์ของฉัน, สร้างอีเวนต์, รายงาน, เช็คอิน, ผู้เข้างาน)
- ฝั่งแอดมิน: 5 หน้า (ภาพรวม, ผู้ใช้, อนุมัติอีเวนต์, การเงิน, คืนเงิน)

> ⚠️ **สำคัญ:** ตอนนี้ข้อมูลทั้งหมดยัง "ปลอม" (mock) — ปุ่มกดได้ หน้าตาเหมือนจริง
> แต่ยัง **ไม่ได้ต่อฐานข้อมูลและระบบล็อกอินจริง** (เป็นงานเฟสถัดไป)

---

## ใช้เทคโนโลยีอะไร

- **Next.js 16** (App Router) — เขียนด้วย **JavaScript (.jsx)** ไม่ใช่ TypeScript
- **Tailwind CSS v4** — ทำหน้าตา
- **Supabase (PostgreSQL)** — ฐานข้อมูล (SQL เขียนไว้แล้วใน `libs/` แต่ยังไม่ได้ต่อ)
- จะใช้ **NextAuth** ทำระบบล็อกอิน (เฟสถัดไป)

ธีม: พื้นขาวมินิมอล สีหลักม่วง `#7c3aed` ฟอนต์ Anuphan (ไทย) + Space Mono (ตัวเลข/โค้ด)

## รันยังไง

```bash
npm install
npm run dev
```

เปิด http://localhost:3000

**ลองดูแต่ละฝั่ง** (พิมพ์ URL ตรงๆ ได้เลย เพราะยังไม่มีระบบล็อกอิน):
- คนซื้อ: `/` แล้วกดซื้อบัตรไปเรื่อยๆ
- คนจัดงาน: `/organizer`
- แอดมิน: `/admin`

---

## ของอยู่ตรงไหน (แผนที่โฟลเดอร์)

```
app/              ← หน้าเว็บทั้งหมด (1 โฟลเดอร์ = 1 หน้า, ไฟล์ page.jsx)
  (shop)/         ← หน้าฝั่งคนซื้อ (มี header ด้านบน)
  organizer/      ← หน้าฝั่งคนจัดงาน
  admin/          ← หน้าฝั่งแอดมิน
components/        ← ชิ้นส่วน UI ที่ใช้ซ้ำ (ปุ่ม, การ์ด, sidebar, ตาราง)
lib/              ← ข้อมูล mock + ฟังก์ชันช่วย (เช่น จัดรูปแบบเงิน ฿)
libs/             ← ไฟล์ SQL ของฐานข้อมูล (ยังไม่ได้ต่อ)
dist/docs/        ← ไฟล์ดีไซน์ต้นฉบับ (.dc.html) ที่เอามาทำตาม
docs/             ← เอกสารสรุปงาน (รวมไฟล์นี้)
```

**เคล็ดลับ:** อยากแก้หน้าไหน ไปหา `page.jsx` ในโฟลเดอร์ชื่อตรงกับ URL
เช่น หน้า `/organizer/report` อยู่ที่ `app/organizer/report/page.jsx`

## คำที่อาจไม่คุ้น (อธิบายสั้นๆ)

- **mock data** = ข้อมูลปลอมที่ใส่ไว้ให้หน้าตาดูมีของ (อยู่ใน `lib/*-mock.js`)
- **placeholder** = ของชั่วคราวที่รอแทนด้วยของจริง (เช่น รูปโปสเตอร์ = กล่องไล่สี, รอใส่รูปจริง)
- **Server / Client Component** = หน้า Next.js มี 2 แบบ; อันที่มีปุ่มกด/โต้ตอบได้จะมี `"use client"` บรรทัดบนสุด
- **route group** เช่น `(shop)` = โฟลเดอร์ในวงเล็บ ใช้จัดกลุ่มหน้า แต่ **ไม่มีผลกับ URL**

---

## ยังเหลืออะไร (เฟสถัดไป)

1. **ระบบล็อกอินจริง** — ซื้อบัตรต้องล็อกอินก่อน, แยกสิทธิ์ 3 ระดับ (คนซื้อ/คนจัด/แอดมิน)
2. **ต่อฐานข้อมูลจริง** — เปลี่ยนข้อมูล mock เป็นข้อมูลจาก Supabase
3. **ใส่รูปจริง + QR จริง** (QR ในตั๋วทำเป็นของจริงแล้ว ส่วนรูปโปสเตอร์ยังเป็น placeholder)
